## Instrucciones para Agentes

**Cuando documentar una mejora:**
1. Implementamos un cambio que mejora el sistema
2. El cambio es medible (más rápido, mejor calidad, menos errores)
3. El cambio es permanente (no temporal)

**Categorías de Mejoras:**

**SEO Improvements:**
- Technical SEO fixes
- Content optimization
- Link building tactics
- Structured data enhancements

**Development Improvements:**
- Code optimizations

- Performance gains

**Process Improvements:**

- Reduced manual steps

**Quality Improvements:**
- Better testing
- Fewer bugs
- Higher consistency
- Better validation

**Velocity Improvements:**
- Faster page creation
- Better templates
- Reduced bottlenecks
- Parallel workflows

## Ejemplo de Buena Improvement Documentation

```markdown
## Improvement #5 - 2026-03-01

**Area**: Content Production
**Type**: Velocity
**Description**: Creado template reutilizable para role pages que reduce tiempo de creación

**Before**:
- Cada role page tomaba 4-6 horas
- Mucho copy-paste manual
- Inconsistencias entre páginas

**After**:
- Template con 16 secciones pre-estructuradas
- Variables para personalización (role, salary, skills)
- Tiempo de creación: <2 horas por página
- 100% consistencia en estructura

**Impact**:
- Velocity: 3x faster (6h → 2h)
- Quality: Consistency score 98% (antes 75%)
- Scale: Ahora podemos producir 10+ pages/week (antes 3-4)
- Cost: Freed up 20+ hours/month

**Source**: Learning #8 - "Templates son clave para escalar sin sacrificar calidad"
```
